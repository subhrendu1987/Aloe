Traditional Internet-of-Things (IoT) relies on the cloud oriented services. Cloud oriented services increase operational expenditure (opex) and due to following reasons.

  1. Cloud lease time is one of the primary opex for IoT infrastructures
  2. The data forwarding cost is also increased for the client as the data requires to be sent to the cloud service provider.
  3. IoT traffics are short and bursty in nature. Therefore, it increases network management overhead.
  4. The infrastructure devices are failure prone which affects the network management policies

In-network processing can avoid some of the problems. In-network processing uses residual resouces of networking devices for data processing.

  1. It Can reduce cloud expenses significantly.
  2. Data forwarding cost and utilization of bottleneck links are reduced in acse of in-network processing.
  3. In-network processing can provide security as the data need not leave the network.
  4. It can provide higher response time in comparison to cloud infrastruce.
  5. However, network management overhead increases as the internal data processing units may migrate from time to time from one device to another device.
  6. Increase in failure probability also affects the total operation.

Architectural Progress

SDN

SDN helps in network mangement by creating centralized view.

d-SDN

Distributed SDN can achieve scalability by deploying multiple instances.

Aloe

Aloe uses μ-controller architecture to exploit in-networking infrastructure.
